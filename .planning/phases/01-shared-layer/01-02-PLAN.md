    Section `# --- ConfigHasher ---`

    **ConfigHasher** — static utility class (no instantiation needed):

    Class-level constants:
    - `TRIGGER_FILES: list[str] = ["config.yaml", "train.py"]` — files that trigger new experiment
    - `TRIGGER_DIRS: list[str] = ["rewards"]` — directories whose `*.py` files trigger new experiment
    - NOTE: `requirements.txt` is intentionally ABSENT from TRIGGER_FILES (per SHRD-09)

    `@staticmethod def _hash_yaml_content(content: bytes) -> str:` — per D-03:
    1. `data = yaml.safe_load(content)` — resolves YAML anchors/aliases
    2. `normalized = json.dumps(data, sort_keys=True, ensure_ascii=True)` — canonical JSON form
    3. Return `hashlib.sha256(normalized.encode("utf-8")).hexdigest()`
    4. Handle `yaml.safe_load` returning None (empty YAML): hash empty string

    `@staticmethod def _hash_python_content(content: bytes) -> str:` — per D-03:
    1. `text = content.decode("utf-8")`
    2. `lines = [line.rstrip() for line in text.splitlines()]` — strip trailing whitespace per line
    3. `normalized = "\n".join(lines)` — normalize line endings to \n
    4. Return `hashlib.sha256(normalized.encode("utf-8")).hexdigest()`

    `@staticmethod def hash_file(path: Path) -> str:` — convenience for single file:
    1. Read `path.read_bytes()`
    2. Dispatch to `_hash_yaml_content` if path.suffix in (".yaml", ".yml"), else `_hash_python_content`
    3. Return the SHA256 hex string

    `@staticmethod def hash_config(scaffold_dir: Path) -> ConfigSnapshot:` — per architecture doc Section 4.2:
    1. Collect trigger files: for each file in `TRIGGER_FILES`, check `scaffold_dir / file` exists; for each dir in `TRIGGER_DIRS`, glob `scaffold_dir / dir / "*.py"` (sorted)
    2. Sort ALL discovered files by relative path (using `sorted()`) — per D-03
    3. Hash each file using `hash_file()`
    4. Build `files: dict[str, str]` mapping relative paths to hashes
    5. Compute `aggregated_hash`: concatenate all hashes in sorted-path order, then `hashlib.sha256(concatenated.encode("utf-8")).hexdigest()`
    6. Return `ConfigSnapshot(snapshot_id=aggregated_hash, files=files, aggregated_hash=aggregated_hash, created_at=datetime.now(tz=timezone.utc))`
    7. Handle empty/missing rewards/ directory gracefully: just skip, no error
    8. Handle missing trigger files gracefully: skip absent files, only hash what exists

    `@staticmethod def diff_snapshots(old: ConfigSnapshot, new: ConfigSnapshot) -> dict[str, tuple[str | None, str | None]]:` — per architecture doc Section 4.2:
    1. Compare `old.files` and `new.files` dictionaries
    2. For files in both: if hashes differ, include `{filename: (old_hash, new_hash)}`
    3. For files only in new: include `{filename: (None, new_hash)}`
    4. For files only in old: include `{filename: (old_hash, None)}`
    5. Return only changed files (empty dict if identical)

    Section `# --- Diff Entry ---`

    **DiffEntry(BaseModel)** — structured diff entry (Claude's discretion: formalized as Pydantic model for type safety):
    - `line: int = Field(..., ge=1, description="Line number in old file (removed) or new file (added)")`

    Section `# --- DiffEngine ---`

    **DiffEngine** — static utility class:

    `@staticmethod def compute_file_diff(old_text: str, new_text: str) -> list[dict[str, int | str]]:` — per architecture doc Section 4.2, research Pattern 5:
    1. Split both texts into lines: `old_lines = old_text.splitlines()`, `new_lines = new_text.splitlines()`
    2. If both empty, return `[]`
    3. Call `difflib.unified_diff(old_lines, new_lines, lineterm="", n=0)` — n=0 means no context lines
    4. Parse output line by line:
       - Skip lines starting with `---` or `+++` (file headers)
       - For lines starting with `@@`: use regex `r"^@@ -(\d+)(?:,(\d+))? \+(\d+)(?:,(\d+))? @@"` with optional count groups (Pitfall 1). Extract old_ln and new_ln starting positions. Subtract 1 from each (will be incremented on first diff line).
       - For lines starting with `-`: increment old_ln, append `{"line": old_ln, "type": "removed", "content": line[1:]}`
       - For lines starting with `+`: increment new_ln, append `{"line": new_ln, "type": "added", "content": line[1:]}`
    5. Return the list of diff entry dicts

    `@staticmethod def compute_scaffold_diff(old_snapshot: "ConfigSnapshot", new_snapshot: "ConfigSnapshot", old_texts: dict[str, str], new_texts: dict[str, str]) -> dict[str, list[dict[str, int | str]] | dict[str, list[dict[str, int | str]]]]:` — per architecture doc Section 4.2:
    1. Import ConfigSnapshot at function level to avoid circular imports (or use TYPE_CHECKING)
    2. Initialize result dict with keys: `config`, `train`, `requirements`, `hyperparams`, `rewards`
    3. Map file categories: `config.yaml` -> key `config`, `train.py` -> key `train`, `requirements.txt` -> key `requirements`
    4. For `config`, `train`, `requirements`: compute `compute_file_diff(old_texts.get(filename, ""), new_texts.get(filename, ""))` and store as `list[dict]`
    5. For `rewards`: iterate over all reward filenames present in either old_texts or new_texts that start with `rewards/`. Build a `dict[str, list[dict]]` keyed by filename (e.g., `"math_reward.py"`: `[...]`). Use just the filename (strip `rewards/` prefix) as the dict key.
    6. `hyperparams` key: always empty list `[]` (hyperparams diff is a downstream concern, placeholder for Phase 4)
    7. Return the complete diff_patch structure
    8. NOTE: requirements.txt IS included in this diff computation per SHRD-09 even though it is excluded from the trigger hash
