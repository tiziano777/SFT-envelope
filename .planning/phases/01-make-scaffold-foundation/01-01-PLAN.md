<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User input → setup_generator | Config file path provided by user; filename used in output directory name |
| Config file → Jinja2 templates | YAML config parsed and passed to template context for rendering |
| Generated files → User environment | Generated Python files executed by user; if templates are corrupted, arbitrary code could run |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | S (Spoofing) | config/loader.py | accept | YAML parsing is deterministic; Pydantic validates schema before generation. Low risk: user controls config source. |
| T-01-02 | T (Tampering) | generators/templates/ | mitigate | Jinja2 templates must be reviewed for code injection vectors. Ensure jinja2.StrictUndefined is set (already line 113 of setup_generator.py). Verify templates don't evaluate user-provided expressions. |
| T-01-03 | R (Repudiation) | tests/test_setup_generator.py | accept | Test output is deterministic. Config + fixture combination produces known output. Audit trail via git commits. |
| T-01-04 | I (Information Disclosure) | setup_generator.py | accept | Generated scaffolds contain user-provided config (model names, dataset names, etc.). User is responsible for keeping this secret if needed. |
| T-01-05 | D (Denial of Service) | setup_generator.py:inject_worker_middleware | mitigate | Middleware copy fails if source doesn't exist (line 50). Error handling is correct (raises FileNotFoundError). Verify middleware/ dir exists before Phase 1 test. |
| T-01-06 | E (Elevation of Privilege) | generators/templates/ | mitigate | Generated train.py and prepare.py are executed by user. Do NOT auto-execute during setup generation. Makefile ensures user must explicitly run generated scripts. |

</threat_model>
