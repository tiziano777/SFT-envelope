## Phase 4 Master API Summary (strategy routing)
- Strategy enum: NEW, RESUME, BRANCH, RETRY
- LineageController.determine_strategy() logic:
  - NEW: No existing experiment with matching config hashes
  - RESUME: Same config hashes, latest checkpoint exists (last_checkpoint_status == "training_done")
  - BRANCH: Config hashes differ but same recipe_id (new training path from different config)
  - RETRY: Same config hashes, latest checkpoint exists (last_checkpoint_status != "training_done" → incomplete)

## Phase 1 Shared Layer (diff/hash utilities)
- ConfigHasher: hash_config(), hash_trigger()
  - trigger_hash() uses: config.yaml + train.py + rewards/* (excludes requirements.txt)
  - Used for branch detection
- DiffEngine: compute_diff() returns list[DiffEntry]
  - Each DiffEntry: line_no, change_type (add/remove/modify), content
  - Used for diff_patch in branch responses

## Existing Tests Pattern (from Phase 4)
- tests/api/test_handshake.py: HandshakeRequest mocking, TestClient usage
- Strategy-specific test cases: NEW (no prior exp), RESUME (same hash + done), BRANCH (diff config), RETRY (same hash + not done)
- Diff assertion: verify DiffEntry objects have correct line numbers and change types

## Test Data Factory Pattern
- conftest.py provides test_config_snapshot fixture
- create_test_handshake_request() in simulate_worker.py for request generation
