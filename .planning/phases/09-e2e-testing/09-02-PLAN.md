## Implementation Tasks

### Task 1: Create `tests/test_handshake.py` — All 4 Strategy Tests

**Goal:** Test all handshake strategies with graph verification

**Files:** `tests/test_handshake.py` (new)

**Test Scenarios:**

#### Scenario 1: NEW Strategy
```
Precondition: No previous experiment with this config hash
Action: Call handshake with config
Expected: strategy=NEW
Verify: 
  - Experiment node created in Neo4j
  - exp_id returned and valid
  - Tag :_TEST applied
```

#### Scenario 2: RESUME Strategy
```
Precondition: Same config hash as before, new checkpoint available
Action: Call handshake with same config, but checkpoint exists
Expected: strategy=RESUME
Verify:
  - Same exp_id returned
  - Latest checkpoint data in response
  - No new experiment created
```

#### Scenario 3: BRANCH Strategy
```
Precondition: Config hash changed (config.yaml or train.py modified)
Action: Call handshake with new config
Expected: strategy=BRANCH
Verify:
  - New exp_id created
  - diff_patch populated with changes
  - DiffEntry line numbers and types correct
  - DERIVED_FROM relationship created
```

#### Scenario 4: RETRY Strategy
```
Precondition: Same config, checkpoint exists, retry counter > 0
Action: Call handshake again
Expected: strategy=RETRY
Verify:
  - Same exp_id returned
  - Same checkpoint returned
  - retry_count incremented in Neo4j
```

**Acceptance Criteria:**
- ✓ test_handshake_new_strategy passes
- ✓ test_handshake_resume_strategy passes
- ✓ test_handshake_branch_strategy passes
- ✓ test_handshake_retry_strategy passes
- ✓ All 4 scenarios verify Neo4j graph mutations
- ✓ All test nodes tagged :_TEST

### Task 2: Create `tests/test_config_change.py` — Trigger Hash & Branch Detection

**Goal:** Verify trigger hash excludes specific files and detects BRANCH correctly

**Files:** `tests/test_config_change.py` (new)

**Tests:**

#### test_config_change_triggers_branch
```
Setup: Run handshake with config_v1
Action: Modify config.yaml (actual content change)
Result: New handshake → strategy=BRANCH
Verify: diff_patch shows changed lines
```

#### test_train_change_triggers_branch
```
Setup: Same config hash baseline
Action: Modify train.py
Result: strategy=BRANCH
Verify: diff_patch contains train.py changes
```

#### test_rewards_change_triggers_branch
```
Setup: Same baseline
Action: Add/remove/modify file in rewards/
Result: strategy=BRANCH
```

#### test_requirements_excluded_from_hash
```
Setup: Handshake with config_v1 (hash H1)
Action: Modify requirements.txt (non-trigger file)
Action: Handshake again with config still v1 but requirements different
Result: strategy=RESUME (NOT BRANCH)

#### test_whitespace_only_change_not_branching
```
Setup: Handshake with config (hash H1)
Action: Add leading spaces to config.yaml (whitespace-only change)
Result: strategy=RESUME (same hash, deterministic)
```

**Acceptance Criteria:**
- ✓ BRANCH triggered on config.yaml changes
- ✓ BRANCH triggered on train.py changes
- ✓ BRANCH triggered on rewards/* changes
- ✓ requirements.txt does NOT trigger BRANCH
- ✓ Whitespace-only changes hashed consistently

### Task 3: Create `tests/test_diff_verification.py` — DiffEntry Correctness

**Goal:** Validate DiffEngine produces correct line-level diffs

**Files:** `tests/test_diff_verification.py` (new)

**Tests:**

#### test_diff_simple_add
```
Old: ["line1", "line2"]
New: ["line1", "line_new", "line2"]
Expected: DiffEntry(line=2, type=ADD, content="line_new")
```

#### test_diff_simple_remove
```
Old: ["line1", "line2", "line3"]
New: ["line1", "line3"]
Expected: DiffEntry(line=2, type=REMOVE, content="line2")
```

#### test_diff_simple_modify
```
Old: ["line1", "old_content", "line3"]
New: ["line1", "new_content", "line3"]
Expected: DiffEntry(line=2, type=MODIFY, content_old="old_content", content_new="new_content")
```

#### test_diff_multiline_file
```
Old: config file with 50 lines
New: config file with 52 lines (2 insertions)
Expected: 2 DiffEntry items with correct line numbers
```

#### test_diff_empty_to_content
```
Old: empty ConfigSnapshot
New: config with content
Expected: DiffEntry items for all lines (type=ADD)
```

#### test_diff_identical_configs
```
Old = New
Expected: empty diff []
```

**Acceptance Criteria:**
- ✓ DiffEntry line numbers are correct (1-indexed)
- ✓ change_type in [ADD, REMOVE, MODIFY]
- ✓ Content preserved exactly (byte-level)
- ✓ Multi-line files handled correctly
- ✓ Empty diffs validated

**Files:** `tests/test_merge_strategy.py` (new)

**Tests:**

#### test_merge_two_checkpoints
```
Setup: Two experiments with checkpoints
Action: POST /merge with [ckp1, ckp2]
Expected: 200, merged checkpoint created
Verify:
  - Merged checkpoint node exists
  - :MERGED_FROM edges to both sources
  - Tag :_TEST applied
```

#### test_merge_three_checkpoints
```
Setup: Three independent experiments
Action: POST /merge with [ckp1, ckp2, ckp3]
Expected: Merged checkpoint with 3 :MERGED_FROM edges

```
Setup: Exp_A merged from Exp_B, Exp_B derived from Exp_A

Setup: Single checkpoint

```

```

```

- ✓ Merge creates correct node with :MERGED_FROM edges
