  <files>workflow.md</files>
  <action>
    Add new subsection "### Worker Daemon Lifecycle in Generated run.sh" after the Phase 16 section.

    Explain the generated run.sh sequence (per run.sh.j2 template):
    1. **Line 1-15: Daemon Bootstrap**
       - Source shared environment
       - Check if daemon already running (prevent double-start)
       - Start worker daemon in background: `python -m worker.daemon &`
       - Save daemon PID to .daemon.pid

    2. **Line 16-25: Handshake Wait**
       - Loop with 30-second timeout (configurable via HANDSHAKE_TIMEOUT)
       - Check for .handshake_done marker file
       - If timeout → WARNING but continue (degraded mode, no lineage sync)
       - If success → log experiment_id from .exp_id file

    3. **Line 26-40: Training Loop**
       - Run user's train.py with all config
       - Daemon watches: lineage/to_transfer/, training_metrics/, config/rewards/
       - Worker pushes checkpoints asynchronously to Master
       - Training proceeds independently of daemon health

    4. **Line 41-45: Training Complete**
       - train.py exits (success or failure)
       - Create .training_done marker file
       - Signal to daemon: flush any pending events

    5. **Line 46-50: Daemon Flush & Cleanup**
       - Wait for daemon to finish flushing transfer_log.jsonl
       - Set daemon one-shot mode: stop after flush completes
       - Cleanup: delete .daemon.pid, .handshake_done (optional, keep for debugging)

    Include ASCII diagram of the flow:
    ```
    time →
    ┌─────────────────────────────────────────────────┐
    │ 1. Daemon       │ 2. Handshake   │ 3. Training   │ 4. Flush & Exit
    │    Bootstrap    │    Wait (30s)  │    Loop       │
    │ ┌─────────────┐ │ ┌───────────┐ │ ┌──────────┐ │ ┌─────────────┐
    │ │daemon start │ │ │check      │ │ │train.py  │ │ │flush        │
    │ │background   │ │ │.handshake │ │ │runs      │ │ │.training_   │
    │ │pidfile      │ │ │_done      │ │ │          │ │ │done marker  │
    │ └─────────────┘ │ │timeout?   │ │ │daemon    │ │ │daemon exit  │
    │                 │ │degraded   │ │ │watches   │ │ │             │
    │                 │ └───────────┘ │ │async push│ │ │             │
    │                 │    .exp_id ✓  │ │          │ │ │             │
    │                 └────────────────┘ └──────────┘ │ │             │
    │                                                  └─────────────────┘
    └─────────────────────────────────────────────────┐
      Worker (GPU) ← async → Master (CPU) lineage store
    ```

    Include Python pseudo-code showing the wait loop:
    ```python
    # Generated run.sh.j2 logic in shell
    for i in $(seq 1 $((HANDSHAKE_TIMEOUT))); do
        if [ -f ".handshake_done" ]; then
            EXP_ID=$(cat .exp_id)
            echo "✓ Handshake complete. Experiment: $EXP_ID"
            break
        fi
        sleep 1
    done
    if [ ! -f ".handshake_done" ]; then
        echo "⚠ Handshake timeout. Running in degraded mode (no lineage sync)"
    fi
    ```

    Note: Daemon runs indefinitely in normal mode, one-shot mode when --one-shot flag is set.
