# Phase 11-01: Core UI Setup & Recipe Management

## Overview

This phase establishes the Streamlit Master UI foundation: project structure, Neo4j async integration (reusing Master API connection patterns), recipe management workflows, and deployment infrastructure. The UI serves as the operational dashboard for LLM fine-tuning experiments.

### F2: Neo4j Async Connection

**Objective:** Reuse Master API async Neo4j client patterns; ensure compatibility with Streamlit's event loop.

**Tasks:**

1. **T2.1:** Create `streamlit_ui/neo4j_async.py` (adapt from `master/neo4j_async.py`)
   - Copy `AsyncNeo4jClient` class
   - Expose methods: `find_recipe_by_name(name: str)`, `list_recipes()`, `get_recipe_details(id: str)`
   - All methods are `async` (Streamlit handles await internally via `asyncio.run`)

   ```python
   from neo4j import AsyncGraphDatabase
   from typing import List, Optional

   class AsyncNeo4jClient:
       def __init__(self, uri: str, user: str, password: str):
           self.driver = AsyncGraphDatabase.driver(uri, auth=(user, password))

       async def find_recipe_by_name(self, name: str):
           async with self.driver.session() as session:
               result = await session.run(
                   "MATCH (r:Recipe {name: $name}) RETURN r",
                   name=name
               )
               return await result.single()

       async def list_recipes(self, limit: int = 100):
           async with self.driver.session() as session:
               result = await session.run(
                   "MATCH (r:Recipe) RETURN r LIMIT $limit",
                   limit=limit
               )
               return await result.list()
   ```

2. **T2.2:** Create `streamlit_ui/storage_client.py`
   - Wrap Master API's `URIResolver` for async artifact checks
   - Expose method: `async check_artifact_exists(uri: str) -> bool`

### F3: Recipe Upload Interface

**Objective:** Enable users to upload YAML recipe configs with client-side validation.

**Tasks:**

1. **T3.1:** Create `streamlit_ui/validation.py`
   - Import `EnvelopeConfig` from `envelope/config/models.py`
   - Define `validate_recipe_yaml(yaml_str: str) -> tuple[bool, Optional[EnvelopeConfig], List[str]]`
   - Return: (is_valid, config_obj, error_list)
   - Validate schema + required fields (name, stage, model, framework)

   ```python
   from envelope.config.models import EnvelopeConfig
   from envelope.config.loader import load_yaml_config
   from pydantic import ValidationError

   def validate_recipe_yaml(yaml_str: str) -> tuple[bool, Optional[EnvelopeConfig], List[str]]:
       try:
           config = load_yaml_config(yaml_str)
           return True, config, []
       except ValidationError as e:
           errors = [f"{err['loc'][0]}: {err['msg']}" for err in e.errors()]
           return False, None, errors
       except Exception as e:
           return False, None, [str(e)]
   ```

2. **T3.2:** Create `streamlit_ui/pages/recipes.py`
   - Add `st.file_uploader("Upload YAML recipe", type=["yaml", "yml"])`
   - On upload: call `validate_recipe_yaml()`
   - If valid: show preview of recipe (name, stage, model, framework, technique list)
   - If invalid: show field-level error messages via `st.error()`
   - Add "Save Recipe" button → POST to Neo4j via `AsyncNeo4jClient.create_recipe()`
   - Show toast on success/failure

   ```python
   import streamlit as st
   from streamlit_ui.validation import validate_recipe_yaml
   from streamlit_ui.utils.caching import get_neo4j_client

   def run():
       st.title("Recipe Management")

       tab1, tab2 = st.tabs(["Upload", "Browse"])

       with tab1:
           uploaded_file = st.file_uploader("Upload YAML recipe", type=["yaml", "yml"])

           if uploaded_file:
               yaml_content = uploaded_file.read().decode("utf-8")
               is_valid, config, errors = validate_recipe_yaml(yaml_content)

               if is_valid:
                   st.success("Recipe validation passed ✓")
                   st.json({"name": config.name, "stage": str(config.stage)})

                   if st.button("Save Recipe"):
                       client = get_neo4j_client()
                       # Async call via asyncio.run
                       result = asyncio.run(client.create_recipe(config))
                       st.toast("Recipe saved!", icon="✓")
               else:
                   st.error("Recipe validation failed")
                   for error in errors:
                       st.error(f"  • {error}")
   ```

## Risks & Mitigations

| Risk | Mitigation |
|------|-----------|
| Streamlit event loop conflicts with async Neo4j | Use `asyncio.run()` wrapper for async calls |
| Session state resets on page navigation | Use `@st.cache_resource` for persistent objects |
| Neo4j connection pooling overhead | Cache client via `@st.cache_resource` (single instance) |
| YAML validation too strict/lenient | Test edge cases (extra fields, type coercion) |
| Docker networking between services | Use docker-compose `depends_on` + `networks` |

## Notes

- Streamlit's native async support is limited; use `asyncio.run()` sparingly (only for page-level operations, not within loops)
- `@st.cache_resource` ensures Neo4j client is created once per session
- Recipe node structure: `Recipe {name, yaml_content, created_at, framework, stage}`
- UI pages follow standard Streamlit patterns: `st.form()`, `st.selectbox()`, `st.dataframe()`
