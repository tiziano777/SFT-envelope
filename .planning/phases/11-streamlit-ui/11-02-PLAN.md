# Phase 11-02: CRUD for Model, Component, Experiment

## Overview

This phase adds CRUD (Create, Read, Update, Delete) interfaces for the three core entities: Model, Component, and Experiment. Each page provides forms with field validation, relationship checks, and integration with Master API PATCH/DELETE endpoints. Delete operations are protected via Neo4j relationship queries to prevent orphaning dependent nodes.

### Entity Models (Neo4j)

```
Model {
  id: UUID
  name: String (unique)
  framework: String (enum: huggingface, keras, pytorch)
  model_size: String (enum: small, medium, large)
  created_at: DateTime
  last_updated: DateTime
}

Component {
  id: UUID
  name: String (unique)
  type: String (enum: optimizer, loss_fn, scheduler, regularizer)
  framework: String (enum: huggingface, keras, pytorch)
  created_at: DateTime
  last_updated: DateTime
}

Experiment {
  id: UUID
  name: String (unique)
  recipe_name: String
  status: String (enum: pending, running, completed, failed)
  created_at: DateTime
  last_updated: DateTime
}

Relationships:
- (Experiment) -[USES_RECIPE]-> (Recipe)
- (Experiment) -[USES_MODEL]-> (Model)
- (Experiment) -[USES_COMPONENTS]-> (Component)
- (Recipe) -[HAS_MODEL]-> (Model)
- (Recipe) -[HAS_COMPONENTS]-> (Component)
```

### F1: Model CRU Page

**Objective:** Provide full CRUD interface for Model entities with framework-specific validation and delete-protection checks.

**Tasks:**

1. **T1.1:** Create `streamlit_ui/crud/model_manager.py`
   - Class `ModelManager`: wraps Neo4j + Master API calls
   - Method `create_model(name, framework, model_size)` → POST to Master API
   - Method `list_models(framework: Optional[str])` → Neo4j query
   - Method `get_model(id: UUID)` → Neo4j query
   - Method `update_model(id, **fields)` → PATCH to Master API
   - Method `check_model_dependencies(id) -> int` → Neo4j relationship count
   - Method `delete_model(id)` → DELETE to Master API (only if 0 dependencies)

   ```python
   class ModelManager:
       def __init__(self, client: AsyncNeo4jClient, api_client: HTTPXClient):
           self.db = client
           self.api = api_client

       async def create_model(self, name: str, framework: str, model_size: str):
           """POST /models"""
           response = await self.api.post("/models", json={
               "name": name,
               "framework": framework,
               "model_size": model_size
           })
           if response.status_code != 201:
               raise UIError(f"Failed to create model: {response.text}")
           return response.json()

       async def check_model_dependencies(self, model_id: str) -> int:
           """Count incoming relationships (Recipe -[HAS_MODEL]-> Model, Experiment -[USES_MODEL]-> Model)"""
           result = await self.db.run(
               "MATCH (m:Model {id: $id})<-[r]-(n) RETURN count(r) as dep_count",
               id=model_id
           )
           return (await result.single())["dep_count"]

       async def delete_model(self, model_id: str):
           """DELETE /models/{id} - protected by dependency check"""
           dep_count = await self.check_model_dependencies(model_id)
           if dep_count > 0:
               raise UIError(
                   f"Cannot delete model: {dep_count} recipe(s) or experiment(s) depend on it",
                   f"Dependencies: Recipe -[HAS_MODEL]-> Model or Experiment -[USES_MODEL]-> Model"
               )
           response = await self.api.delete(f"/models/{model_id}")
           if response.status_code != 204:
               raise UIError(f"Failed to delete model: {response.text}")
   ```

   ```python
   import streamlit as st
   from streamlit_ui.crud.model_manager import ModelManager
   from streamlit_ui.utils.caching import get_neo4j_client
   from streamlit_ui.api_client import HTTPXClient

   def run():
       st.title("Model Management")

       client = get_neo4j_client()
       api_client = HTTPXClient(st.session_state.config.master_api_url)
       manager = ModelManager(client, api_client)

       tab_create, tab_browse, tab_edit, tab_delete = st.tabs([
           "Create", "Browse", "Edit", "Delete"
       ])

       # CREATE TAB
       with tab_create:
           st.subheader("Create New Model")
           with st.form("create_model_form"):
               name = st.text_input("Model Name", placeholder="gpt2-large")
               framework = st.selectbox("Framework", ["huggingface", "keras", "pytorch"])
               model_size = st.selectbox("Model Size", ["small", "medium", "large"])
               submitted = st.form_submit_button("Create Model")

               if submitted:
                   if not name.strip():
                       st.error("Model name is required")
                   else:
                       try:
                           result = asyncio.run(manager.create_model(name, framework, model_size))
                           st.success(f"Model '{name}' created successfully!")
                       except UIError as e:
                           st.error(f"{e.user_message}: {e.details}")

       # BROWSE TAB
       with tab_browse:
           st.subheader("Browse Models")
           framework_filter = st.selectbox("Filter by Framework",
               ["All Frameworks", "huggingface", "keras", "pytorch"])

           filter_val = None if framework_filter == "All Frameworks" else framework_filter
           models = asyncio.run(manager.list_models(framework=filter_val))

           if models:
               df = pd.DataFrame([
                   {
                       "Name": m["name"],
                       "Framework": m["framework"],
                       "Size": m["model_size"],
                       "Created": m["created_at"]
                   }
                   for m in models
               ])
               st.dataframe(df, use_container_width=True)
           else:
               st.info("No models found.")

       # EDIT TAB
       with tab_edit:
           st.subheader("Update Model")
           models = asyncio.run(manager.list_models())
           model_names = [m["name"] for m in models]

           selected_model_name = st.selectbox("Select Model to Edit", model_names)

           if selected_model_name:
               model = asyncio.run(manager.get_model(selected_model_name))
               with st.form("edit_model_form"):
                   model_size = st.selectbox(
                       "Model Size",
                       ["small", "medium", "large"],
                       index=["small", "medium", "large"].index(model["model_size"])
                   )
                   submitted = st.form_submit_button("Update Model")

                   if submitted:
                       try:
                           asyncio.run(manager.update_model(model["id"], model_size=model_size))
                           st.success("Model updated!")
                       except UIError as e:
                           st.error(f"{e.user_message}")

       # DELETE TAB
       with tab_delete:
           st.subheader("Delete Model")
           models = asyncio.run(manager.list_models())
           model_names = [m["name"] for m in models]

           selected_model_name = st.selectbox("Select Model to Delete", model_names)

           if selected_model_name:
               model = asyncio.run(manager.get_model(selected_model_name))
               dep_count = asyncio.run(manager.check_model_dependencies(model["id"]))

               if dep_count > 0:
                   st.warning(f"⚠️ This model is used by {dep_count} recipe(s)/experiment(s). Cannot delete.")
               else:
                   st.info("✓ No dependencies found. Safe to delete.")
                   confirm = st.checkbox(f"I confirm deletion of '{selected_model_name}'")
                   if confirm and st.button("Delete Model"):
                       try:
                           asyncio.run(manager.delete_model(model["id"]))
                           st.success("Model deleted!")
                       except UIError as e:
                           st.error(f"{e.user_message}")
   ```

### F3: Experiment CRU Page

**Objective:** Provide CRUD interface for Experiment entities with automatic relationship linking (Recipe → Model, Component).

**Tasks:**

1. **T3.1:** Create `streamlit_ui/crud/experiment_manager.py`
   - Methods: `create_experiment(name, recipe_name, status)`, `list_experiments()`, `get_experiment()`, `update_experiment()`, `delete_experiment()`
   - `create_experiment()` auto-links relationships:
     - Query Recipe node by name
     - Extract Model ID from Recipe's HAS_MODEL relationship
     - Extract Component IDs from Recipe's HAS_COMPONENTS relationships
     - Create (Experiment) -[USES_RECIPE]-> (Recipe), -[USES_MODEL]-> (Model), -[USES_COMPONENTS]-> (Component)
   - Delete protection: check for checkpoint dependencies (Experiment -[HAS_CHECKPOINT]-> Checkpoint)

   ```python
   class ExperimentManager:
       async def create_experiment(self, name: str, recipe_name: str, status: str):
           """Create experiment and auto-link to recipe's model + components"""
           # 1. Get Recipe node
           recipe = await self.db.run(
               "MATCH (r:Recipe {name: $name}) RETURN r",
               name=recipe_name
           )
           recipe_node = await recipe.single()

           if not recipe_node:
               raise UIError(f"Recipe '{recipe_name}' not found")

           # 2. POST /experiments
           response = await self.api.post("/experiments", json={
               "name": name,
               "recipe_name": recipe_name,
               "status": status
           })
           if response.status_code != 201:
               raise UIError(f"Failed to create experiment: {response.text}")

           exp_data = response.json()
           exp_id = exp_data["id"]

           # 3. Auto-link relationships in Neo4j
           await self.db.run(
               """
               MATCH (e:Experiment {id: $exp_id}), (r:Recipe {name: $recipe_name})
               CREATE (e)-[rel:USES_RECIPE]->(r)
               """,
               exp_id=exp_id,
               recipe_name=recipe_name
           )

           # 4. Link to Recipe's Model
           await self.db.run(
               """
               MATCH (e:Experiment {id: $exp_id}), (r:Recipe {name: $recipe_name})-[hm:HAS_MODEL]->(m:Model)
               CREATE (e)-[rel:USES_MODEL]->(m)
               """,
               exp_id=exp_id,
               recipe_name=recipe_name
           )

           # 5. Link to Recipe's Components
           await self.db.run(
               """
               MATCH (e:Experiment {id: $exp_id}), (r:Recipe {name: $recipe_name})-[hc:HAS_COMPONENTS]->(c:Component)
               CREATE (e)-[rel:USES_COMPONENTS]->(c)
               """,
               exp_id=exp_id,
               recipe_name=recipe_name
           )

           return exp_data

       async def check_experiment_dependencies(self, exp_id: str) -> int:
           """Check for checkpoint relationships"""
           result = await self.db.run(
               "MATCH (e:Experiment {id: $id})-[r:HAS_CHECKPOINT]->(cp) RETURN count(r) as dep_count",
               id=exp_id
           )
           return (await result.single())["dep_count"]
   ```

### F4: Master API Integration

**Objective:** Create HTTPXClient wrapper for PATCH/DELETE calls to Master API.

**Tasks:**

1. **T4.1:** Create `streamlit_ui/api_client.py`
   - Class `HTTPXClient`: async wrapper around httpx.AsyncClient
   - Methods: `post()`, `patch()`, `delete()` (all include X-API-Key header)
   - Handle retries: exponential backoff for transient errors (500, 503)
   - Raise `UIError` on 4xx/5xx responses

   ```python
   import httpx
   from streamlit_ui.errors import UIError

   class HTTPXClient:
       def __init__(self, base_url: str, api_token: str):
           self.base_url = base_url
           self.api_token = api_token
           self.client = httpx.AsyncClient(base_url=base_url)

       async def patch(self, endpoint: str, json: dict):
           headers = {"X-API-Key": self.api_token}
           response = await self.client.patch(endpoint, json=json, headers=headers)

           if response.status_code >= 400:
               raise UIError(
                   f"API error: {response.status_code}",
                   response.text
               )
           return response

       async def delete(self, endpoint: str):
           headers = {"X-API-Key": self.api_token}
           response = await self.client.delete(endpoint, headers=headers)

           if response.status_code >= 400:
               raise UIError(
                   f"API error: {response.status_code}",
                   response.text
               )
           return response
   ```

### F5: Delete Protection Pre-Checks

**Objective:** Prevent orphaning dependent nodes by querying relationships before delete.

**Tasks:**

1. **T5.1:** Implement in each Manager class
   - `check_X_dependencies(id)` → returns count of incoming relationships
   - Called before DELETE endpoint
   - If count > 0: raise `UIError` with user-friendly message
   - UI shows warning + disables delete button

   Example Neo4j queries:
   - Model: `MATCH (m:Model {id: $id})<-[r]-(n) RETURN count(r) as dep_count`
   - Component: `MATCH (c:Component {id: $id})<-[r]-(n) RETURN count(r) as dep_count`
   - Experiment: `MATCH (e:Experiment {id: $id})-[r:HAS_CHECKPOINT]->(cp) RETURN count(r) as dep_count`

2. **T5.2:** Update pages to show dependency warnings
   - Delete tab: always call `check_X_dependencies()` before rendering form
   - If count > 0: show `st.warning()` + disable delete button
   - If count = 0: show `st.success()` + enable delete button

## Risks & Mitigations

| Risk | Mitigation |
|------|-----------|
| Circular dependencies in auto-linking | Use Neo4j transactions; validate Recipe exists before creating Experiment |
| Race condition: Delete called while dependencies exist | Include dependency check in DELETE endpoint (Master API layer) |
| Stale Neo4j data in dropdowns | Add refresh button; use `@st.cache_resource` with short TTL |
| Missing relationship links | Transaction wrapper in `create_experiment()` ensures all 4 relationships created atomically |
| API timeouts on large datasets | Add pagination; limit list queries with `LIMIT 100` |

## Notes

- Experiment auto-linking happens in 2 layers: Master API creates Experiment node, then UI creates relationships in Neo4j
- Consider adding Neo4j transactions to ensure atomic auto-linking
- Recipe preview shows data from Recipe node (model name extracted from HAS_MODEL relationship)
- Delete protection checks should mirror delete endpoint logic (both query relationship counts)
- Consider caching recipe/model/component lists with 5-minute TTL to reduce Neo4j queries
