# Phase 11-03: Integration, Deployment & Testing

## Overview

This phase integrates the Streamlit UI with the Master API and Neo4j deployment infrastructure. It updates docker-compose.yml to orchestrate all services, adds Makefile convenience commands, implements end-to-end integration tests that verify the full UI → API → Database call chain, and ensures health checks pass across all services.

## Architecture

### Service Mesh (Docker Compose)

```yaml
services:
  neo4j:
    - Port 7687 (Bolt)
    - Port 7474 (HTTP)
    - Port 7473 (HTTPS)

  master:
    - Port 8000 (FastAPI)
    - Depends on: neo4j

  streamlit:
    - Port 8501 (Web UI)
    - Depends on: master, neo4j
    - Network: envelope (custom bridge)
```

```

   @pytest.mark.asyncio
   async def test_e2e_model_lifecycle(neo4j_client, api_client):
       """E2E: Create model → Edit → Delete"""
       manager = ModelManager(neo4j_client, api_client)

       # 1. Create
       model = await manager.create_model("e2e_gpt2", "huggingface", "small")
       assert model["model_size"] == "small"

       updated = await manager.update_model(model["id"], model_size="large")
       assert updated["model_size"] == "large"

       dep_count = await manager.check_model_dependencies(model["id"])
       assert dep_count == 0

       await manager.delete_model(model["id"])
       models = await manager.list_models()
       assert not any(m["id"] == model["id"] for m in models)

   async def test_e2e_model_delete_protected(neo4j_client, api_client):
       """E2E: Cannot delete model if recipe depends on it"""
       model_manager = ModelManager(neo4j_client, api_client)
       recipe_manager = RecipeManager(neo4j_client, api_client)

       # 1. Create model
       model = await model_manager.create_model("protected_gpt2", "huggingface", "small")

       # 2. Create recipe using model
       recipe = await recipe_manager.create_recipe_with_model("recipe_1", model["id"])

       # 3. Try to delete model → should fail
       dep_count = await model_manager.check_model_dependencies(model["id"])
       assert dep_count > 0

           await model_manager.delete_model(model["id"])
       assert "Cannot delete model" in str(exc_info.value)

   @pytest.mark.asyncio
   async def test_e2e_experiment_auto_linking(neo4j_client, api_client):
       """E2E: Create experiment → verify auto-link to recipe's model/components"""
       recipe_manager = RecipeManager(neo4j_client, api_client)
       exp_manager = ExperimentManager(neo4j_client, api_client)

       # 1. Create recipe with model + components
       recipe = await recipe_manager.create_recipe_full(
           name="full_recipe",
           model_id="model_1",
           component_ids=["comp_1", "comp_2"]
       )

       # 2. Create experiment
       exp = await exp_manager.create_experiment("exp_1", recipe["name"], "pending")

       # 3. Verify auto-linked relationships
       result = await neo4j_client.run(
           "MATCH (e:Experiment {id: $id})-[r:USES_MODEL]->(m:Model) RETURN m.id",
           id=exp["id"]
       )
       linked_model = await result.single()
       assert linked_model["m.id"] == "model_1"

       result = await neo4j_client.run(
           "MATCH (e:Experiment {id: $id})-[r:USES_COMPONENTS]->(c:Component) RETURN count(c) as cnt",
           id=exp["id"]
       )
       comp_count = (await result.single())["cnt"]
       assert comp_count == 2
   ```

## Risks & Mitigations

| Risk | Mitigation |
|------|-----------|
| Docker network DNS resolution fails | Use service hostnames (neo4j, master, streamlit) in compose; verify bridge network created |
| Health checks timeout on slow systems | Increase timeout from 5s to 10s; add retry buffer in conftest |
| Tests fail due to stale data between runs | Fixture `docker_stack` clears DB on teardown; use unique test data prefixes |
| Port conflicts (8000, 8501, 7687 in use) | Use docker-compose port mapping; document host port requirements |
| Async test fixtures deadlock | Use `pytest-asyncio` with `asyncio_mode = auto` |
| Coverage report incomplete | Add `--cov-config=.coveragerc` for granular control |

## Notes

- All services communicate via internal docker-compose network (envelope bridge)
- Health checks ensure graceful degradation: if Neo4j unhealthy, Master API waits; if Master API unhealthy, Streamlit waits
- E2E tests assume docker-compose stack running; conftest fixture handles setup/teardown
- Test data uses unique prefixes (e.g., `e2e_*`, `test_*`) to avoid collisions
- Coverage target: 80%+ for streamlit_ui package
- Consider adding load testing via pytest-benchmark for API endpoints
